# SATD Self-Repayment Prediction - Experimental Scripts and Results

This package contains the experimental scripts and results for the temporal validation of SATD (Self-Admitted Technical Debt) self-repayment prediction models.

## Directory Structure

- `scripts/` - Python scripts for data processing, model training, and analysis
- `results/` - Experimental results (CSV, JSON, Markdown reports)
- `figures/` - Generated figures (PDF/PNG)
- `data/` - Dataset files (not included in repository, see below)

## Data Files

The following dataset files are required to run the scripts (not included in this repository due to size):

| File | Size | Description | Source Location |
|------|------|-------------|-----------------|
| `merged_data_40501_extended.json` | 156MB | Main dataset with 55 features | `dataset/data2/` |
| `merged_data_40501_temporal_sorted_by_add_date.json` | 128MB | Time-sorted dataset for temporal validation | `dataset/data2/` |
| `merged_data_40501_phase1.json` | 127MB | Intermediate dataset with phase 1 features | `dataset/data2/` |
| `merged_data_40501_phase2.json` | 129MB | Intermediate dataset with phase 2 features | `dataset/data2/` |
| `satd.db` | 953MB | SQLite database for identity label sensitivity analysis | Generated by script `26` |

**Note**: Place these files in a `data/` directory at the same level as `scripts/` before running the scripts. The `satd.db` file is only needed for script `26_identity_label_sensitivity.py`.

### Data Generation

These dataset files were generated from the original downloaded dataset. If you have the raw data, you can regenerate them using:

```bash
# Scripts 10 and 14 generate intermediate datasets
python scripts/10_compute_phase1_features.py
python scripts/14_compute_extended_features.py
```

Input: The original SATD dataset (40,501 instances with features)
Output: Processed datasets with engineered features for temporal validation

### Text Model Results

**Note on naming**: Script `05_run_temporal_text_models.py` has default output names:
- `temporal_text_model_metrics.csv`
- `temporal_text_model_summary.md`

The actual results in `results/` use different names to distinguish between two experimental protocols:

**10-90 protocol** (9 expanding-window splits, 10%-90% training):
```bash
python scripts/05_run_temporal_text_models.py \
  --metrics-csv results/temporal_text_model_metrics_10_90_full.csv \
  --summary results/temporal_text_model_summary_10_90_full.md \
  --protocol fixed-test
```

**80-20 protocol** (single 80/20 split):
```bash
python scripts/05_run_temporal_text_models.py \
  --metrics-csv results/temporal_text_model_metrics_80_full.csv \
  --summary results/temporal_text_model_summary_80_full.md \
  --train-percentages 80
```

Results files:
- `temporal_text_model_metrics_10_90_full.csv` / `temporal_text_model_summary_10_90_full.md`
- `temporal_text_model_metrics_80_full.csv` / `temporal_text_model_summary_80_full.md`

## Reproducibility Notes

1. **Random seeds**: All scripts use fixed random seeds (typically 42 or 2025) for reproducibility
2. **API keys**: Script `22` requires `GITHUB_TOKEN` and `DEEPSEEK_API_KEY` for live API calls; cached results are already provided in `validation_commit_messages_raw.json`
3. **GPU training**: Text models (script `05`) support CUDA; CPU fallback is available but slower
